Repo drilldown query review

Runtime and merge evidence for PR 152.

Reported failure

GET /api/repo-drilldown returned ClickHouse NO_COMMON_TYPE.

There is no supertype for types String, DateTime
while executing greater(toString(__table1.hour), _CAST(..., 'DateTime'))

The aggregate query projected toString(hour) AS hour while the same identifier remained visible to the filter and grouping expressions.

Fix

The aggregate branch now aliases the source column to bucket_hour inside a bounded subquery. Filtering and rollup grouping stay DateTime-safe; only the final output field is formatted as hour.

hour AS bucket_hour
...
GROUP BY bucket_hour WITH ROLLUP

The merge with main also preserves the newer per-actor activity-feed migration, the older activity-feed fallback, and a shared high-water anchor.
